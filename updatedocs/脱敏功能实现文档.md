# 脱敏功能实现文档

## 1. 功能介绍

脱敏功能是 WeKnora 系统中的一个安全特性，用于在文档分块前对敏感信息进行替换，保护用户隐私和敏感数据。该功能允许用户配置敏感词和对应的替换词，系统会在解析文档后、分块前自动对敏感词进行替换处理。

### 主要功能点：
- 支持启用/禁用脱敏功能
- 支持配置多个敏感词和对应的替换词
- 支持在文档分块前自动应用脱敏处理
- 支持父子分块模式下的脱敏处理

## 2. 实现逻辑

脱敏功能的实现分为前端配置和后端处理两个部分：

### 2.1 前端配置
1. 在知识库分块设置界面添加脱敏功能开关
2. 提供敏感词和替换词的输入界面
3. 将配置保存到知识库的 ChunkingConfig 中

### 2.2 后端处理
1. 在文档解析完成后，分块前应用脱敏处理
2. 对所有文本内容进行敏感词替换
3. 支持普通分块和父子分块两种模式的脱敏

## 4. 让功能生效的步骤

### 4.1 后端部署

#### 4.1.1 Scripts 目录挂载（重要）
在修改后端代码后，需要通过挂载 scripts 目录来实现代码热更新。操作步骤如下：

1. **首次配置时需要从容器拷贝 scripts 目录到本地**：
   ```bash
   # 先停止容器
   docker-compose down app
   
   # 启动容器（不挂载 scripts）
   docker-compose up -d app
   
   # 等待容器启动后，从容器拷贝 scripts 目录到本地
   docker cp WeKnora-app:/app/scripts/. ./scripts/
   
   # 停止容器
   docker-compose down app
   ```

2. **修改 docker-compose.yml 添加 scripts 挂载**：
   在 `docker-compose.yml` 的 app 服务 volumes 配置中添加：
   ```yaml
   volumes:
     # 后端源码挂载（只读，用于参考）
     - ./:/app_src:ro
     # 脚本目录挂载（可写，用于修改 entrypoint 脚本）
     - ./scripts:/app/scripts:ro
     # 其他挂载...
   ```

3. **重启容器使挂载生效**：
   ```bash
   docker-compose up -d app
   ```

**注意**：scripts 目录必须先从容器中拷贝出来，再进行挂载。否则容器启动时会因为缺少必要的脚本文件而无法正常启动。

#### 4.1.2 代码修改和部署
1. 确保后端服务正在运行
2. 修改代码后，容器会自动检测源文件变化并重新编译（通过 scripts/docker-entrypoint.sh 脚本实现）
3. 如果自动编译失败，可以手动重启容器：`docker restart WeKnora-app`

### 4.2 前端部署
1. 确保前端代码已构建：在 `frontend` 目录中运行 `npm run build` 命令
2. 重启前端容器：`docker restart WeKnora-frontend`

### 4.2 前端配置
1. 进入知识库设置页面
2. 点击「分块设置」选项卡
3. 找到「脱敏功能」开关，将其开启
4. 在「敏感词」输入框中输入需要脱敏的词语，多个词语用逗号分隔
5. 在「替换为」输入框中输入对应的替换词，多个替换词用逗号分隔，位置与敏感词一一对应
6. 点击保存按钮，保存配置

## 5. 使用说明

### 5.1 配置示例
**敏感词**：密码,身份证,手机号
**替换为**：***,****,*****

### 5.2 处理流程
1. 用户上传文档
2. 系统解析文档内容
3. 应用脱敏处理，将敏感词替换为指定的替换词
4. 对处理后的文本进行分块
5. 生成向量嵌入并存储

### 5.3 注意事项
- 敏感词和替换词的数量必须一致
- 敏感词匹配是精确匹配，区分大小写
- 脱敏处理会在分块前应用，确保所有分块内容都经过脱敏处理
- 脱敏处理不会修改原始文档，只在分块过程中应用

## 6. 测试验证

1. 配置敏感词和替换词
2. 上传包含敏感信息的测试文档
3. 查看分块结果，确认敏感信息已被替换
4. 进行检索测试，确认检索结果中不包含敏感信息

## 7. 故障排查

### 7.1 脱敏功能不生效
- 检查脱敏功能是否已开启
- 检查敏感词和替换词配置是否正确
- 检查前端是否已重新构建并部署
- 检查后端服务是否正常运行
- 检查后端 API 处理层是否正确处理脱敏配置

### 7.2 脱敏配置保存失败
- 检查后端 `internal/handler/initialization.go` 文件中是否正确添加了脱敏配置处理
- 检查 KBModelConfigRequest 结构体是否包含 SensitiveConfig 字段
- 检查 UpdateKBConfig 函数是否正确更新脱敏配置
- 检查 buildConfigResponse 函数是否正确返回脱敏配置

### 7.3 脱敏效果不符合预期
- 检查敏感词是否正确配置
- 检查敏感词是否区分大小写
- 检查文档中是否包含完全匹配的敏感词

## 8. 总结

脱敏功能是 WeKnora 系统中的重要安全特性，通过在文档分块前对敏感信息进行替换，有效保护了用户隐私和敏感数据。该功能实现了前端配置与后端处理的完整流程，支持灵活的敏感词配置，为用户提供了安全可靠的知识库管理体验。
